## 1. 產品概述

**Study Anywhere Voyage（SAV）** 是一款為台灣上班族設計的每日英文學習應用程式，透過以真實生活場景為基礎的結構化每日練習，幫助使用者提升英文對話能力，尤其是聽力與口說。

產品名稱的首字母（S.A.V.）同時也是創辦人的名字縮寫（Savelyn）。品牌核心精神：

> 學習，從不受限於任何地方；就像一段航行，風浪不會讓你停下，只會讓你學會如何掌舵前行。

英文版：
> *Study anywhere. Like a voyage at sea, the rough waves don't stop you — they teach you to navigate.*

### 核心價值主張

- **有結構但有彈性** — 完整的 365 天課程，從每位使用者的 Day 1 開始往後滾動，不綁定固定上線日或固定曆年
- **主題連貫，不孤立** — 每週的聽力、口說和複習內容圍繞同一主題，透過重複接觸強化詞彙
- **真實世界的語言** — 內容以真實對話情境為基礎，非教科書式的練習題

### Prototype 現況

已有一份完整可操作的 HTML prototype，作為所有功能、UI 和內容結構的標準參考。開發者應以 prototype 作為 UX 行為與視覺設計的最終依據。

## 2. 目標使用者

### 主要 Persona

- **人口特徵：** 台灣上班族、20歲以上
- **英文程度：** 中級（B1–B2）— 可閱讀技術文件，但難以自然地用英文表達
- **痛點：**
  - 說話前先在腦中翻譯，造成語言延遲
  - 沒有持續的練習習慣
  - 現有 App 的詞彙與日常工作/生活脫節
- **每日可用時間：**
  - 通勤 60–80 分鐘（上下班）
  - 午休 10–15 分鐘
  - 睡前 5–10 分鐘

### 隱私原則

所有 Podcast 內容使用**虛構角色**（主角＋訪問者，每日隨機人物名稱），不使用任何真實人物資訊。內容中不得出現可識別個人身份的地名、職稱、裝備品牌或生活細節。

## 3. 學習哲學

### 每日三段式學習循環

| 時段 | 活動 | 時長 | 目標 |
|------|------|------|------|
| 早晨通勤 | Listen（Podcast 對話） | 10 分鐘 | 被動輸入、詞彙接觸 |
| 午休 / 下班後 | Conversation（口說提示） | 10–15 分鐘 | 主動輸出、AI 批改 |
| 睡前 | Review（字卡複習） | 5–10 分鐘 | 記憶鞏固、間隔重複 |

### 每日三模組 × 每週同主題

每天都有三個模組，圍繞同一個週主題從不同角度切入：
- **每天的 Listen** — 全新 Podcast 集數，探索該週主題的一個面向（每週 7 集）
- **每天的 Conversation** — 當天口說提示，從個人角度回應主題（每週 7 題）
- **每天的 Speak** — 當天朗讀文章，延伸閱讀主題相關議題（每週 7 篇）
- **週日的 Review** — 字卡複習當週累積的核心詞彙

### 難度進程（六階段，W1–W53）

課程採**滾動式 365 天**設計，從用戶第一次開始使用的當天算起（Day 1），不綁定固定曆年。
課程起始日寫入使用者的 `profiles.settings.curriculumStartDate`，之後所有週次與日期標籤都依此動態推算。

| 階段 | 週次 | 主題 |
|------|------|------|
| 日常生活 | W1–W10 | 重新開始、晨間例行、通勤、家居、標記與傳統、飲食、天氣、購物、健康、時間安排 |
| 人際關係 | W11–W18 | 友誼、家庭、同事、社交、個性、溝通、助人、衝突解決 |
| 工作與職涯 | W19–W26 | 工作職責、會議、截止日、問題解決、職涯目標、學習成長、成功與失敗、工作生活平衡 |
| 興趣與生活 | W27–W34 | 旅行、攝影、音樂、閱讀、寵物、嗜好、自然、運動與健身 |
| 觀點與視野 | W35–W43 | 日常科技、AI、心理健康、環境、金錢、改變、價值觀、未來、回顧前進 |
| 深度探索 | W44–W53 | 創意、領導力、社群回饋、跨文化、語言認同、休息更新、感恩、目標設定、年末回顧、新的開始 |

## 4. 功能模組

### 4.1 Listen（聆聽）

**目的：** 透過英文對話腳本進行被動聽力練習  
**使用情境：** 早晨通勤，10 分鐘  
**分頁代表色：** 藍色 `#7AB8E8`  
**Tab icon：** 播放三角形 SVG

#### 功能列表

- **Podcast 對話播放器** — 雙角色英文對話腳本（每集約 8.5–9.5 分鐘），TTS 自動逐行播放
- **集數標題列** — 自動根據使用者自己的課程進度顯示 Day 編號、週次、主題與 Podcast 標題；可輔助顯示實際日期，但內容不綁定固定曆年日期
- **逐行腳本顯示** — 腳本渲染為可點擊行；點擊任一行可跳轉至該行播放
- **中文翻譯切換** — `中文 OFF / 中文 ON` 切換每行中文翻譯
- **詞彙卡片** — 當前播放行展開時顯示生字，附中文釋義
- **播放速度控制** — 0.75×、1.0×、1.25×
- **懸浮迷你播放器** — 捲動超過主播放器時右下角出現，可繼續控制播放
- **進度指示器** — 當前行數 / 總行數
- **關鍵句型面板** — 集數底部整理的高價值句型

#### 腳本角色設定（六階段輪替）

為增加收聽多樣性，每個學習階段更換一組角色名稱，但人物定位不變：
- **主角**（說話者 A）— 上班族，分享日常生活與工作經驗
- **訪問者**（說話者 B）— 好奇的朋友，提問引導對話

| Phase 2 | W11–W18 | Lily | Tom |
| Phase 3 | W19–W26 | Sara | Alex |
| Phase 4 | W27–W34 | Nina | Marcus |
| Phase 5 | W35–W43 | Jade | Ryan |
| Phase 6 | W44–W53 | Maya | James |

- 說話者顏色：主角用暖米白（`--ui`），訪問者用柔和綠灰（`#8AB4A0`）

#### 內容規範

- 每集約 1,200 字，6 個 parts，48 行對話
- 每集 8–10 個詞彙項目
- **嚴禁**使用真實地名、品牌、個人識別資訊

#### 內容規模

- 全年共 **365 集**（W1/W53 各 4 集，其餘 51 週各 7 集）
- 每集約 1,200 字，6 個 parts，48 行對話

### 4.2 Conversation（口說練習）

**目的：** 透過書面英文提示進行主動輸出並獲得 AI 批改  
**使用情境：** 午休或下班後，10–15 分鐘  
**分頁代表色：** 紫色 `#9B84C4`  
**Tab icon：** 對話框 SVG

#### 功能列表

- **每日問題卡片** — 顯示今日口說題目、中文提示、句型結構提示
- **當週主題標籤** — 顯示當週主題作為問題背景脈絡
- **答案輸入區** — 使用者以英文輸入回答
- **AI 批改（點數制）** — 扣除點數後送至 Claude API，以繁體中文回傳結構化批改
- **空白防呆** — 未輸入就送出時，輸入框邊框閃爍提示

#### AI 批改格式

```
1. 整體評價（1–2 句，先給正向肯定）
2. 1–2 個改進點（附修正範例）
3. 一個更道地的說法
```

## 5. 內容架構

### Podcast 集數（Listen）

每週一個主題，每天一集，全年 365 集。內容檔以**週為單位**組織（`week-01.ts` ~ `week-53.ts`），每檔包含該週所有每日集數陣列。

```
週模組（week-XX.ts）
└── Episode[]（陣列，W1/W53 有 4 個，其餘週有 7 個）
    ├── weekNumber: number       // 1–53
    ├── dayOfWeek: number        // 1–7（1=週一）
    ├── date?: string            // legacy/reference only；不得作為內容選取依據
    ├── theme: string            // 當週主題
    ├── title: string            // 當集標題
    ├── phase: string            // p1–p6
    ├── parts: EpisodePart[]     // 5 段落
    │   └── lines: EpisodeLine[] // 35–45 行對話
    │       ├── speaker: 'a' | 'b'
    │       ├── speakerName: 'Mira' | 'Jamie'
    │       ├── en: string
    │       ├── zh: string
    │       └── vocab?: VocabItem[]
    └── keyPhrases: KeyPhrase[]  // 8–10 個
```

### 朗讀文章（Speak）

```
文章
├── 週次與 dayOfWeek（rolling curriculum lookup key）
├── 主題（7 類之一）
├── 文章標題
├── 英文正文（HTML，5 段落，約 600 字）
├── 中文翻譯（HTML，段落對應）
└── 詞彙（4–6 個，參照 CEFR B2）
```

## 6. 每日課表設計

### 每日節奏

每天都有三個模組，不分平日或週末：

| 時段 | 活動 | 時長 |
|------|------|------|
| 早晨通勤 | **Listen** — 當天 Podcast 集數（全新，非重播） | 10 分鐘 |

| 天 | Listen 集數方向 |

| Day 1 | 主題總覽與核心對話 |
| Day 2 | 個人經驗與日常場景 |
| Day 3 | 挑戰與解決方案 |
| Day 4 | 不同觀點與比較 |
| Day 5 | 深入探討與延伸 |
| Day 6 | 實際應用與建議 |
| Day 7 | 反思與總結 |

## 9. 商業模式

### App 下載定價

| 平台 | 定價 | 開發者實拿（小開發者方案 15%） |
|------|------|-------------------------------|
| Apple App Store | NT$60 | ~NT$51 |
| Google Play Store | NT$60 | ~NT$51 |

> **小開發者方案：** Apple 與 Google 均提供年收入 < $1,000,000 USD 的開發者申請 15% 抽成（預設 30%）。**上架前務必申請**，免費且立即生效。

### 帳號系統與登入方式

v1 包含最小化帳號系統（點數制必要）。支援三種登入方式：

| 方式 | 說明 | 備註 |
|------|------|------|
| Email + 密碼 | 傳統登入 | 必選 |
| Sign in with Apple | Apple ID 一鍵登入 | **Apple 強制要求**（只要提供其他第三方登入即必做）|
| Google Sign In | Google 帳號一鍵登入 | Android 用戶體驗最順 |

### IAP 付款整合

點數購買必須透過 Apple In-App Purchase 與 Google Play Billing（App Store 規定，不得使用第三方金流）。使用 **RevenueCat** 作為 IAP 代理層，統一處理雙平台、收據驗證與 Webhook 通知。

### AI 批改計費：點數制

Conversation 的 AI 批改功能採用點數制（Credit System）：

| 方案 | 點數 | 定價 | 每次批改成本 |
|------|------|------|------------|
| 入門包 | 100 點 | NT$99 | ~NT$1.0 |
| 標準包 | 300 點 | NT$249 | ~NT$0.83 |
| 重度包 | 600 點 | NT$399 | ~NT$0.67 |

**設計原則：**
- 每次送出答案前顯示「剩餘點數」
- 點數不足時引導購買，不強制中斷學習
- 點數**無到期日**（降低購買壓力）
- 後端扣點必須使用原子操作，防止重複扣點
