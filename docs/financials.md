## 二、一次性建置成本

| 項目 | 費用 | 備註 |
|------|------|------|
| Apple Developer Program | NT$3,400 | 已付，每年續費 |
| OpenAI TTS 音檔生成（全年） | ~NT$1,230（~$38.45 USD） | 依新版長篇內容量估算：365 集、17,520 行、2,563,318 chars；採 tts-1 |
| **合計** | **NT$4,630** | |

> **決策備註：** 已實測比較 `tts-1` 與 `tts-1-hd` 的 W1D1 sample，聽感差異不明顯，因此正式版本以較低成本的 `tts-1` 為準。
>
> **分批策略：** 若要降低前期現金支出，可先產 Week 1–4，再視用戶留存補產後續週次；但目前產品內容已改為全年長篇版，成本口徑應以新版內容量為準。

## 四、每筆交易成本（每賣出一包 10 Credits）

| 項目 | 金額 |
|------|------|
| 售價 | NT$60 |
| Apple 抽成 30% | -NT$18 |
| **開發者實收** | **NT$42** |
| Claude Haiku（10 次批改） | -NT$1.44（$0.0045 USD × 10） |
| **每包毛利** | **NT$40.56** |
| **毛利率（對實收）** | **96.6%** |

> Claude Haiku 每次批改費用明細：
> Input ~300 tokens × $0.80/MTok + Output ~300 tokens × $4.00/MTok ≈ $0.00144 USD ≈ NT$0.046
>
> ⚠️ 抽成假設：本表用 Apple 30%。**Apple 小開發者方案 15% 已申請、待審核**（見 SAV-dev-progress.md Owner TODO）。一旦核准，實收升至 NT$51、每包毛利約 NT$49.56，下方損益數字會更好。為與 backend-hosting-decision.md 對齊，本文暫以保守的 30%（毛利 NT$40.56）為基準。

## 七、成本結構優勢

1. **邊際成本極低** — 每增加一個用戶，額外成本僅 Claude API 費用（NT$0.046/次批改），幾乎為零
2. **後端成本固定且極低** — Supabase Pro $25/月（NT$800/月）為固定成本，不隨用戶數成長（音檔由 Cloudflare R2 服務、egress 免費）；即使 10,000 MAU 後端仍 $25/月
3. **無訂閱壓力** — 買斷 + 消耗型點數，用戶心理門檻低
4. **高毛利率** — 實收毛利 96.6%，優於大多數實體或 SaaS 產品

## 八、主要財務風險

| 風險 | 說明 | 因應 |
|------|------|------|
| 用戶數不足 | 月活 < 7 人則無法回收年費 | App Store 搜尋優化、口碑 |
| Apple 審查退件 | 延誤上線影響早期用戶獲取 | 確保 Guideline 2.1 合規 |
| 後端成本上漲（低機率） | 已採 Supabase Pro + R2（egress 固定 $0），MAU 破萬仍不漲 | 見 backend-hosting-decision.md 的重評門檻 |
| RevenueCat 超額 | 月營收超過 $2,500 USD（≈ NT$80,000）時需付費 | 這是好問題，屆時再評估 |
