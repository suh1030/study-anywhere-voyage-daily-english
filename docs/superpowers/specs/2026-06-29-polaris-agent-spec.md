## 1. 問題陳述

Notch Up! 的自學者在 365 天課程中獨自練習，遇到三個問題：

- **練了不知道對不對**：造句/口說沒有即時、可信的糾錯與更道地說法。
- **看不到自己的狀態**：想問「我到第幾天了？漏了哪天？字卡練得如何？」但沒有對話式入口。
- **孤獨、易放棄**：缺乏一個親切、隨時可問的「引路人」維持動機。

**目標**：提供一個全 App 可用的 AI 英文家教「Polaris」，能即時糾錯、個人化回答學習狀態、鼓勵持續練習——且在成本受控（免費模型）下，**回答的事實絕不出錯**。

> 證據等級：低。上述痛點為假設，尚未有訪談/數據佐證（見 §9）。

## 2. 使用者與使用情境

**主要 Persona — 「每天想進步一點的自學者」**
中級（B1–B2）台灣英文學習者，用 Notch Up! 做每日練習；會用中英混雜提問；在意「我今天有沒有做、做得對不對」；對隱私敏感。

**核心 User Stories（含驗收標準）**


| #   | 作為使用者，我想…        | 驗收標準（Given / When / Then）                                                                         |
| --- | ---------------- | ------------------------------------------------------------------------------------------------- |
| US1 | 讓 Polaris 幫我糾正英文 | Given 我送出有文法錯的句子；When Polaris 回覆；Then 明確指出錯誤、給正確版本、用繁中簡短解釋。若句子本來就對，先肯定「這樣沒錯」，再（選擇性）給更道地說法，不得誤稱有錯。 |
| US2 | 問我目前的學習進度        | Given 我今天還沒完成；When 我問進度；Then 回報「今天尚未完成、完成率 X%」，數字與後端一致；查不到時誠實說看不到；**絕不編造數字或完成狀態**。                |
| US3 | 剛完成就想確認          | Given 我剛在 Schedule 劃完今天；When 我回聊天問進度；Then 必須反映「今天已完成」。                                            |
| US4 | 問有哪些字卡可練習        | Given 我問可練習的字卡；When Polaris 回覆；Then 只列出後端回傳的真實字卡，**不得自行發明單字**。                                    |
| US5 | 認識這個 App 與家教     | Given 我請 Polaris 介紹 App 或它自己；Then 用 2–4 句準確說明功能與角色，不誇大不存在的功能、也不婉拒。                                |
| US6 | 不被帶偏題            | Given 我要求寫程式／長篇翻譯／聊時事；Then 禮貌婉拒，並把它轉成一個英文練習機會。                                                    |
| US7 | 危機時被妥善對待         | Given 我表達自傷或緊急醫療風險；Then 先給同理與立即求助建議，**不把危機轉成英文練習**。                                               |
| US8 | 得到一致、安全的回應       | Given 任意輸入（含越獄嘗試）；Then 一律台灣繁體中文、不洩漏內部指示、不因使用者要求而改變角色或捏造事實。                                        |

## 3. 功能需求

系統**必須**：

- **FR1** 提供全 App 右下角浮動按鈕開啟的多輪對話聊天室。
- **FR2** 對英文造句提供即時糾錯與更道地說法（US1）。
- **FR3** 回答學習進度（第幾天、完成率、漏掉哪天、今天是否完成），數字一律來自後端權威資料（US2、US3）。
- **FR4** 回答字卡精熟數與「可練習字卡」清單，僅限真實資料（US4）。
- **FR5** 準確介紹 App 與家教自身（US5）。
- **FR6** 將無關任務婉拒並導回英文練習（US6）。
- **FR7** 對安全/醫療危機優先給予求助建議（US7）。
- **FR8** 全程使用台灣繁體中文、不洩漏內部指示、不被越獄（US8）。
- **FR9** 免費不扣點，但每人每日訊息數有上限並顯示剩餘額度。

## 4. Agent 行為規格

### 4.1 意圖 → 動作對應

系統須依使用者**語意**（非關鍵字）判斷並採取下列其一：


| 意圖                   | 動作          |
| -------------------- | ----------- |
| 問學習表現／進度／今天是否做完／漏掉哪天 | 查後端進度資料再回答  |
| 問可學/複習/練習的單字、要單字清單   | 查後端可練習字卡再回答 |
| 問已精熟幾張、總共幾張          | 查後端字卡統計再回答  |
| 文法/時態/句型/翻譯/一般英文教學   | 直接回答，不需個人資料 |


### 4.2 事實可信度（硬需求）

- 進度與字卡等事實的**唯一可信來源是後端唯讀查詢**；前端一律不可作為事實來源。
- 回答中的數字必須來自後端回傳；否則系統須**攔截並改由其他方式重試**，寧可不給數字也不得捏造。
- 「今天是否完成」須與後端一致，不得出現「未完成卻說已完成」或反向錯誤。

### 4.3 拒絕與安全行為

- 越獄、竄改進度、要求洩漏內部指示 → 拒絕並維持家教角色。
- 代寫可直接繳交的評分作業 → 拒絕，改為引導/修改草稿。
- 跨使用者資料 → 拒絕且不編造。
- 自傷/緊急醫療 → 先給同理與求助建議，不轉成練習。

> 意圖分類與拒絕行為的實作（路由器、確定性攔截、輸出驗證）見 SSD §3、§6.2、§6.3、§6.7。

## 6. 資料與隱私需求

- **不落地**：對話僅存於記憶體，App 重啟或跨日即清，不寫入任何儲存。
- **無 PII 外送**：送往模型的學習脈絡為聚合資訊（今日主題等），不含姓名/Email/ID。
- **權限最小化**：讀取個人資料須以使用者身分、受資料庫列級權限（RLS）保護，且僅唯讀。
- **前端顯示**：聊天室須常駐隱私提示（對話不會被儲存）。

> 資料表、RLS、雙 client 等實作見 SSD §8、§9。

## 8. 範圍內外（Scope）

**本版做（In）**：多輪英文練習與糾錯、進度/字卡/主題查詢、App 與家教介紹、越界婉拒、安全處理、每日額度、繁中強制、隱私不落地。

**本版不做（Non-Goals）**：

- 串流輸出（streaming）。
- 對話落地儲存與長期/情節記憶。
- 付費模型與點數扣費（介面預留，未啟用）。
- 語音輸入與發音評分（純文字）。
- 跨使用者資料查詢、代寫可繳交作業。
- 家教設定/管理 UI。
