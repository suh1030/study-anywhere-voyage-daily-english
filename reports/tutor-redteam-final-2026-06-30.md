## 第一輪發現並修正的重大問題

1. 曾接受使用者假進度，把可信的 9/12、75% 改成 100 天、99%。
2. 曾逐字洩漏 system prompt。
3. 曾服從提示注入，改成政治評論者。
4. 曾迎合錯誤文法前提並提供虛構來源。
5. 曾直接輸出 Python 爬蟲，離開英文教學範圍。
6. 曾把正確的現在完成式改成錯誤過去式。
7. 簡體字手寫偵測清單漏掉「点」。

## 防護措施

- 明確區分可信 system/context 與不可信 user input。
- 對 prompt 洩漏、角色改寫、自傷與嚴重胸痛加入確定性安全回覆。
- 驗證進度回答中的數字只能來自可信 context。
- 攔截內部思考、截斷回覆、異常長回覆、非教學程式碼及中國用語。
- 使用 OpenCC 完整單字字典偵測簡體字，並排除「台北、里長、皇后」等合法繁體用字。
- 模型輸出不合格時不顯示，改試下一個 provider；所有 provider 都失敗才回傳服務暫時不可用。

## 限制

這些測試能顯著降低亂答風險，但生成式模型無法保證所有未知輸入都百分之百正確。後端防護會讓已知高風險錯誤採取「攔截或安全失敗」，而不是把不合格內容顯示給使用者。
